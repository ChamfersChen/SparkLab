## 目标

为 SparkLab 搭建一个可运行、可热重载、可测试的最小骨架，**不包含具体业务实现**。骨架完成后，任何业务模块（认证、模板管理、工作流……）都可以无障碍接入。

成功标准：

1. `docker compose up -d` 后所有服务健康（api-dev / postgres / redis），并能本机 `pnpm dev` 跑起来前端
2. 访问 `GET /api/system/health` 返回 `{"status":"ok"}`
3. 访问 `http://localhost:5173/` 看到 Vue 应用挂载成功（即便页面是空的）
4. 后端目录边界（`server` HTTP 层 + `sparklab` 业务包）成型，可以直接加业务路由
5. 前端 `apis/base.js` 走 Bearer Header 鉴权（与 Yuxi 一致），可以直接加业务 API
6. Alembic 初始化完成，有一个空的 baseline 迁移
7. `pytest` 能跑通一个最小的 health-check e2e 测试

## 已确认的技术决策（2026-06-25）

| 项 | 决策 |
|---|---|
| 资讯 `status` 枚举 | **三态**：`draft` / `published` / `archived`（与模板、工作流一致） |
| 鉴权方式 | **JWT + `Authorization: Bearer` Header**（localStorage 存储），见 [[sparklab-auth-bearer-header]] |
| 数据库迁移 | **Alembic**（每次 schema 变化生成迁移文件） |
| 密码强度 | 最小 6 位（与产品文档 §8.2 一致，后期可强化） |
| 数据库 | 直接使用 Postgres（asyncpg + SQLAlchemy 异步） |
| 前端运行位置 | **本机 `pnpm dev`**（端口 5173），不再用 Docker 容器；`docker-compose.yml` 中 `web-dev` 服务已注释 |
| API 宿主机端口 | **5151**（容器内仍是 5050，通过 compose 映射） |
| API 镜像基底 | `FROM base-api:0.1.0`（项目主理人维护的自定义基底，含系统依赖与 uv） |
| 首页定位 | **public 欢迎页**：未登录可访问，Hero CTA 按登录态切换（登录 ↔ 进入系统），下方为业务模块入口卡（骨架阶段全 disabled），见 [[sparklab-homepage-is-welcome-page]] |

## 关键不变量（搭建过程中必须遵守）

来自 [[sparklab-dev-constraints]] memory 和 `AGENTS.md`：

1. 后端鉴权用 **`Authorization: Bearer` Header**（与 Yuxi 一致；前端 localStorage 存储 token）
2. 路由层薄、领域厚：`server` 只做 HTTP 适配，业务逻辑在 `sparklab` 包
3. 前端 API 必须集中在 `web/src/apis`，组件不许散落拼 URL
4. 颜色一律用 `base.css` CSS 变量（主色 `--main-color: #3B82F6`）
5. 不允许过度防御/回退掩盖设计缺陷

## 关键技术决策

1. **草稿存储位置**：`localStorage`，key 形如 `sparklab:template-draft:{id}`。不上后端：草稿是用户本地未完成输入，跨设备同步不是 MVP 范围。
2. **localStorage 写入策略**：`watch(formValues, persistDraft, { deep: true })`，输入即存；切换页面 / 刷新均不丢失。
3. **打开 AI 平台** = 先复制 prompt，再 `window.open(url, '_blank')`。原因：浏览器跨域 URL 预填 prompt 被安全策略阻断；让用户手动 Ctrl/⌘+V 是当前唯一可行方案。
4. **URL 状态同步**：用 `router.replace({ query })` 而非 `push`，避免每次筛选都污染浏览器历史栈。
5. **`ref` vs `reactive`**：保持项目惯例用 `ref({})` + `.value`，规避 ESLint 现有配置对 `<script setup>` 中 `reactive` 导入识别的 pre-existing 缺陷。

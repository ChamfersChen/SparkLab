## 目标

为管理员提供后台「数据看板」入口 `/admin/dashboard`，直观看到平台整体使用情况：

- 核心指标卡：模板/工作流/用户/激活码/累计使用/累计收藏
- 模板 Top10 排行（按使用次数）
- 工作流 Top10 排行（按使用次数）
- 激活码分布饼图
- 使用趋势折线图
- 近期发布动态时间线

支持时间范围筛选：**近 7 天 / 近 30 天 / 全部**（顶部 Segmented 切换）。

## 关键决策（已确认）

| 项 | 决定 |
|---|---|
| 图表库 | **ECharts**（`echarts` + `vue-echarts`）— 满足柱状/折线/饼图需求 |
| 时间范围筛选 | **7d / 30d / all** 三段切换 |
| 资讯项 | 本次不包含（资讯模块未实现） |
| 近期动态数据源 | 复用现有 `templates.created_at` / `playbooks.created_at`（不建 activity_log） |
| 文件命名 | 后端 `dashboard_*` 系列；前端 view 文件 `AdminDashboardView.vue` |

## 范围

### 在范围

- 后端 4 个 API（stats / top-templates / top-playbooks / recent-activity）
- 时间范围语义：
  - **all**：`templates.use_count` / `playbooks.use_count` 累计求和；`favorites` 全部计数
  - **7d / 30d**：基于 `template_runs.created_at` / `playbook_runs.created_at` / `favorites.created_at` 范围统计
- 前端 `/admin/dashboard` 页面：6 张指标卡 + 趋势折线 + 激活码饼图 + 2 张排行柱状图 + 近期动态时间线

### 不在范围

- 资讯统计（等资讯模块上线后补）
- 自定义日期范围（仅预设 7d/30d/all）
- 实时刷新（手动切换时间范围触发）
- 单个用户视角的看板（仅全平台总览）

## 后端设计

### API 清单

| 方法 | 路径 | 功能 | 权限 |
|---|---|---|---|
| GET | `/api/admin/dashboard/stats` | 核心指标 + 激活码分布 | 管理员/超管 |
| GET | `/api/admin/dashboard/top-templates` | 模板 Top N（按 use_count） | 管理员/超管 |
| GET | `/api/admin/dashboard/top-playbooks` | 工作流 Top N（按 use_count） | 管理员/超管 |
| GET | `/api/admin/dashboard/recent-activity` | 近期发布动态（最近 20 条） | 管理员/超管 |

### Query 参数

- `range`: `7d` | `30d` | `all`（默认 `all`），适用于 stats / top-* 三个端点
- `limit`: 仅 top-* 接受，默认 10，上限 50
- recent-activity 不接受 range，只取最近 20 条 published 状态

### 指标语义

| 指标 | all 模式 | 7d/30d 模式 |
|---|---|---|
| `templates_published` | 状态=published 的模板数（不变） | 同左 |
| `playbooks_published` | 状态=published 的工作流数（不变） | 同左 |
| `total_uses` | sum(`templates.use_count`) + sum(`playbooks.use_count`) | count(`template_runs`) + count(`playbook_runs`) |
| `total_favorites` | count(`favorites`) | count(`favorites`) where created_at 在区间内 |
| `users_total` | count(users)（不变） | 同左 |
| `users_new` | —（省略） | count(users) where created_at 在区间内 |
| `activation_codes` | {unused, used, disabled, total}（不变） | 同左 |

## 风险与注意

- **使用次数累计与区间统计差异**：`all` 走 `use_count` 字段，区间走 `runs` 表计数，文档需说明（看板顶部可加 tooltip 解释）
- **ECharts 包大小**：~200KB，dashboard 单页使用可接受；按需引入（`echarts/core` + 注册组件）控制 size
- **时区**：后端用 `datetime.now(UTC)` 算区间边界；前端展示按本地时区
- **趋势图空态**：当区间内无数据时显示空图 + 「暂无数据」提示

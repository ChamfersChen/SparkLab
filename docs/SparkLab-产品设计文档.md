# SparkLab · 产品设计文档 v3

> **定位**：管理员发布「提示词模板」和「模板工作流 Playbook」 → 付费用户持激活码激活账号 → 按工作流步骤逐步填空生成提示词 → 复制到第三方 AI 平台使用
>
> **核心体验**：用户要的不是一堆孤立模板，而是一个完整流程——按工作流步骤一步一步执行，每步生成一个提示词，走完流程拿到一套完整结果
>
> **不做的事**：不内置 AI 生成、不做付费会员体系（但预留扩展空间）、不开放公开注册、不做用户评论/评分/社区、不做工作流变量自动传递（极简版：每步独立）

---

## 1. 角色与权限

| 角色 | 可见入口 | 可执行操作 |
|---|---|---|
| **超级管理员 (super_admin)** | 用户端全部 + 后台管理全部（含管理员账号、激活码、AI 平台配置） | 全部权限：模板/工作流/标签管理 + 激活码 + 账号管理 + 平台配置 |
| **管理员 (admin)** | 用户端全部 + 模板管理、工作流管理、标签管理、数据看板 | 发布/编辑/下线模板和工作流；管理标签 |
| **普通用户 (user)** | 用户端（首页/Dashboard、模板库、模板详情、填写页、工作流、收藏、个人中心） | 浏览、收藏、使用模板和工作流生成提示词 |

> **设计原则**：管理员之间不做内容归属隔离，任何管理员都可以编辑全部已发布或草稿内容。编辑操作记录操作人，便于追溯。

## 2. 信息架构（站点地图）

### 用户端（普通用户可访问）
1. **激活页** `/activate?code=xxx` → 验证激活码，设置账号
2. **登录页** `/login` → 用户名/密码登录
3. **首页 / Dashboard** `/` → 推荐工作流 + 热门模板 + 快捷入口
4. **模板库** `/templates` → 搜索 + 三维标签筛选 + 模板卡片网格
5. **模板详情页** `/templates/:id` → 模板说明 + 需填写信息 + 示例效果 + 使用次数 + "填写信息生成提示词"主按钮
6. **提示词填写页** `/templates/:id/fill` → 左表单 / 右 Prompt 预览（可编辑）/ 复制 / 第三方 AI 跳转
7. **工作流列表** `/playbooks` → 所有可用工作流卡片网格
8. **工作流运行页** `/playbooks/:id` → Step 列表 + 每步独立填写流程
9. **我的收藏** `/favorites` → 收藏的模板卡片列表
10. **个人中心** `/profile` → 用户名/头像 / 修改密码 / 退出登录

### 后台管理（管理员 + 超级管理员共用）
13. **模板管理** `/admin/templates` → 列表 + 创建/编辑/发布/下线 + variable_hints 配置
14. **工作流管理** `/admin/playbooks` → 列表 + 创建/编辑工作流（添加步骤、关联模板、排序）
15. **标签管理** `/admin/tags` → 三维标签（平台/内容类型/行业场景）的增删改 + 排序
16. **激活码管理** `/admin/codes` → 仅超管可见 → 生成激活码 / 查看列表 / 禁用/启用
17. **管理员账号管理** `/admin/admins` → 仅超管可见 → 邀请/移除/升级管理员
18. **第三方 AI 平台配置** `/admin/platforms` → 仅超管可见 → 维护跳转平台列表
19. **数据看板** `/admin/dashboard` → 基础统计 + 模板/工作流使用排行 + 近期发布动态

> **导航规则**：左侧固定导航。未登录用户仅看到登录入口。已登录普通用户看到用户端入口（首页 / 模板库 / 工作流 / 我的收藏 / 个人中心）。管理员额外看到后台管理入口。超级管理员在后台管理中额外看到激活码管理、管理员账号管理、平台配置。

## 3. 核心用户旅程

### 旅程 A：用户首次使用（激活 + 登录）

```
用户收到管理员发送的激活链接：https://sparklab.com/activate?code=KL7X-92A3-BM4R
    ↓
点击链接，浏览器打开网站
    ↓
前端检测到 URL 中的 code 参数，发送给后端验证
    ↓
    ┌─ 激活码有效且未使用 → 显示"设置账号"页面（用户名 + 密码）
    │                       ↓
    │                     用户填写信息 → 提交
    │                       ↓
    │                     后端：标记激活码为"已使用" + 创建用户 + 返回 JWT Token
    │                       ↓
    │                     前端：跳转到首页 / Dashboard（已登录状态）
    │
    └─ 激活码无效/已使用/已禁用 → 显示"激活码无效"提示页面
```

### 旅程 B：用户第二次及以后使用

```
用户访问 https://sparklab.com
    ↓
前端检查 localStorage 中是否有有效 Token
    ↓
    ├─ 有 Token 且后端验证通过 → 直接进入首页 / Dashboard ✅
    │
    └─ 无 Token 或 Token 过期 → 显示登录页，用户输入用户名/密码登录
```

### 旅程 C：用户使用工作流 Playbook 完成创作任务（核心体验）

```
用户进入首页 / Dashboard
    ↓
看到"推荐工作流"区块，展示 3-6 个工作流卡片
    ↓
用户点击某个工作流 → 进入工作流运行页
    ↓
看到工作流简介 + Step 列表：
    Step 1：XXX         [开始]
    Step 2：YYY         [待 Step 1 完成]
    Step 3：ZZZ         [待 Step 2 完成]
    ↓
用户点击 Step 1 的"开始"按钮 → 进入该步骤的提示词填写页
    ↓
左栏显示该步骤模板的表单（变量填写提示）→ 用户填写 → 点击"生成提示词"
    ↓
右栏显示拼接好的完整 Prompt → 用户可编辑调整 → 点击"复制提示词"
    ↓
用户点击第三方 AI 平台快捷按钮 → 新标签页打开平台 → 粘贴 Prompt → 获取 AI 生成结果
    ↓
用户点击"完成此步骤" → Step 1 标记为"已完成"，Step 2 的"开始"按钮解锁
    ↓
用户点击 Step 2 的"开始" → 重复上述流程（每步独立，变量不自动传递）
    ↓
...直到完成所有步骤
    ↓
用户获得一套完整的创作产出
```

### 旅程 D：用户使用单模板生成提示词

```
用户在模板库浏览 → 点击某模板 → 进入模板详情页
    ↓
查看模板说明、需填写信息清单、示例效果
    ↓
点击"填写信息生成提示词" → 进入提示词填写页
    ↓
左侧表单自动生成输入框（根据模板中 {{变量名}} 自动识别 + variable_hints 提示）
    ↓
用户填写表单 → 点击"生成提示词"
    ↓
右侧展示拼接完成的完整 Prompt（支持用户在线编辑调整）
    ↓
点击"复制提示词" → 内容复制到剪贴板
    ↓
点击底部的第三方 AI 平台快捷按钮 → 新标签页打开对应平台
    ↓
用户在第三方平台粘贴 Prompt 使用
```

### 旅程 E：管理员发布模板

```
管理员进入「模板管理」→ 点击"新建模板"
    ↓
填写标题、一句话描述
    ↓
按五段式结构填写 Prompt 模板内容：
  Role（角色定义）
  Goal（目标说明）
  Input（变量定义，使用 {{变量名}} 语法）
  Output（输出要求）
  Example（示例效果）
    ↓
配置三维标签（平台 / 内容类型 / 行业场景，多选）
    ↓
保存为草稿 或 直接发布
    ↓
发布后，模板出现在用户端模板库中
```

### 旅程 F：管理员创建工作流 Playbook

```
管理员进入「工作流管理」→ 点击"新建工作流"
    ↓
填写工作流基本信息：
  - 标题（必填）
  - 一句话描述（必填）
    ↓
添加步骤（按顺序添加）：
  Step 1：选择模板「XXX」→ 填写 Step 名称
  Step 2：选择模板「YYY」→ 填写 Step 名称
  ...
    ↓
保存为草稿 或 直接发布
    ↓
发布后，工作流出现在用户端的工作流列表和首页推荐中
```

## 4. 页面详细说明

### 4.1 激活页 `/activate?code=xxx`

**功能**：验证 URL 中的激活码，引导用户设置账号

**页面逻辑**：
1. 页面加载时，立即读取 URL 中的 `code` 参数
2. 发送给后端验证：`POST /api/activation/verify`
3. 根据验证结果显示不同内容：
   - **激活码有效且未使用**：显示"设置账号"表单（用户名 + 密码 + 确认密码），用户提交后创建账号并自动登录
   - **激活码已使用**：提示"该激活码已被使用，请联系管理员获取新的激活码"，提供"去登录"按钮
   - **激活码无效/不存在/已禁用**：提示"无效的激活码，请检查链接是否正确，或联系管理员"
4. 提交账号后：`POST /api/activation/activate` → 后端创建用户 + 返回 JWT Token → 前端写入 localStorage → 跳转首页

### 4.2 登录页 `/login`

**功能**：已注册用户通过用户名密码登录

**页面内容**：
- 用户名输入框
- 密码输入框
- "记住我"复选框（勾选则 Token 有效期 30 天，不勾选则浏览器关闭即失效）
- "登录"按钮
- "还没有账号？联系管理员获取激活码"文字提示

**页面逻辑**：
1. 用户提交 → `POST /api/auth/login`
2. 验证成功 → 返回 JWT Token → 前端写入 localStorage → 跳转到首页
3. 验证失败 → 显示"用户名或密码错误"

### 4.3 首页 / Dashboard `/`

**功能**：用户登录后的入口页，展示核心功能概览，引导用户进入工作流或模板库

**页面内容**：
- 顶部欢迎区："欢迎回来，{用户名}！"
- **推荐工作流**区块（主视觉，占据页面上半部分）：
  - 标题："推荐工作流"
  - 3-6 个工作流卡片网格
  - 每张卡片展示：工作流名称、一句话描述、步骤数量、使用次数
  - 点击卡片直接进入工作流运行页
- **热门模板**区块（次要视觉，页面下半部分）：
  - 标题："热门模板"
  - 4-8 个模板卡片网格
  - 每张卡片展示：模板名称、一句话描述、使用次数
  - 点击卡片进入模板详情页
- 底部快捷入口："查看全部工作流"（跳转 /playbooks）、"查看全部模板"（跳转 /templates）

**空状态**：暂无数据时，展示引导提示和"去工作流管理"的管理员快捷入口（管理员可见）

### 4.4 模板库 `/templates`

**功能**：用户浏览和查找模板的完整列表页

**页面内容**：
- 顶部：搜索框（支持按模板标题、描述关键词搜索）
- 筛选区：三维标签筛选器（标签值由管理员动态配置）
  - **平台**：下拉多选
  - **内容类型**：下拉多选
  - **行业/场景**：下拉多选
- 排序：按"使用次数（从高到低）"默认排序，可切换"最新发布"
- 模板卡片网格：
  - 标题
  - 一句话描述（截断显示）
  - 标签（平台 / 内容类型 / 行业场景，显示前 2-3 个）
  - 使用次数
  - "收藏"图标按钮（已收藏显示为实心，未收藏显示为空心）
  - 点击卡片 → 进入模板详情页

**空状态**：搜索/筛选无结果 → "没有找到匹配的模板，换个关键词试试"

### 4.5 模板详情页 `/templates/:id`

**功能**：让用户在动手填写前判断模板是否适用

**页面内容**：
- 顶部：模板标题
- 标签区：平台标签、内容类型标签、行业/场景标签
- 一句话描述
- "需要填写的信息"区块：
  - 列出所有变量名（从 Input 段的 `{{变量名}}` 自动提取）
  - 每个变量名下方展示一段简短的填写提示（从 `variable_hints` JSON 字段中读取；如管理员未填写该变量的提示，则仅显示变量名，不展示额外提示）
  - 示例展示：
    ```
    • 产品名称
      请输入产品完整名称，包括品牌名
    • 核心卖点
      列出2-3个最吸引用户的卖点，用逗号分隔
    ```
- "示例效果"区块：展示 Example 段内容（管理员在创建模板时填写的理想输出样例）
- 使用次数显示
- "填写信息生成提示词"主按钮（大按钮，显眼位置）
- "收藏"按钮

**页面逻辑**：
- 点击主按钮 → 跳转至提示词填写页 `/templates/:id/fill`
- 访问模板时，`使用次数 +1`（同一用户多次访问同一模板，只记一次或每次都记？按次记，简单处理）

### 4.6 提示词填写页 `/templates/:id/fill`

**功能**：把填表转换成一段可直接使用的完整提示词

**页面布局（左右两栏）**：

**左栏 — 结构化表单**：
- 页面顶部显示模板名称，可点击返回模板详情页
- 表单字段根据模板 Input 段中的 `{{变量名}}` 自动生成输入框
- 每个输入框上方显示变量名（作为字段标题）
- 每个输入框下方显示变量填写提示（从 `variable_hints` 中读取；如未配置则不显示）
- 输入框类型：默认多行文本框（textarea），方便用户写较长内容
- "生成提示词"按钮

**右栏 — Prompt 预览区**：
- 标题："生成的提示词"
- 可编辑的文本区域（textarea 或可编辑 div），展示拼接完成的完整 Prompt
- 拼接规则：五段式按顺序输出（Role → Goal → Input（替换变量为用户填写内容）→ Output → Example）
- 用户可以直接在预览区编辑、调整、补充 Prompt 内容
- "复制提示词"按钮（大按钮，显眼位置）→ 点击后 Toast 提示"已复制到剪贴板"
- 第三方 AI 平台快捷按钮组（按钮内容由超管在"第三方 AI 平台配置"中维护）→ 点击在新标签页打开对应平台

**页面逻辑**：
1. 用户填写左侧表单 → 点击"生成提示词"
2. 前端执行本地拼接逻辑（不请求后端）：
   - 读取模板五段式内容
   - 将 Input 段中的 `{{变量名}}` 替换为用户填写的实际内容
   - 按 Role / Goal / Input / Output / Example 顺序拼接为完整文本
   - 每段之间用空行或分隔线分隔，保持可读性
3. 拼接结果填充到右侧预览区
4. 用户可以在预览区直接编辑
5. 用户点击"复制提示词"→ 复制当前预览区内容到剪贴板
6. 用户点击第三方 AI 平台按钮 → 新标签页打开对应平台 URL

### 4.7 工作流列表 `/playbooks`

**功能**：用户浏览所有可用工作流

**页面内容**：
- 顶部：搜索框（支持按工作流名称、描述搜索）
- 工作流卡片网格：
  - 工作流名称
  - 一句话描述
  - 步骤数量（如"4 步流程"）
  - 使用次数
  - 点击卡片进入工作流运行页
- 排序：按使用次数（从高到低）默认排序，可切换"最新发布"

**空状态**：暂无工作流时，展示引导提示（管理员可见"去创建工作流"快捷入口）

### 4.8 工作流运行页 `/playbooks/:id`

**功能**：用户按步骤完成多步骤工作流，每步独立生成提示词

**页面内容**：
- 顶部：工作流名称、一句话描述、步骤数量
- **Step 列表区域**（按顺序排列）：
  - 每个 Step 卡片展示：
    - Step 序号（如 "Step 1"）
    - Step 名称（如 "账号定位分析"）
    - Step 简要说明（如管理员填写，则显示；否则不显示）
    - 状态按钮：
      - 第一个 Step：显示"开始"按钮（绿色可点击）
      - 后续 Step：当前面步骤未完成时，显示"待上一步完成"（灰色不可点击）
      - Step 完成后：显示"已完成"标记（绿色）和"重新操作"按钮（允许用户重新进入）
- **Step 详情区**（当用户点击某个 Step 的"开始"后，在页面右侧或弹窗中显示）：
  - 显示该 Step 关联的提示词填写页（左表单、右预览，与独立模板填写页一致）
  - Step 内的提示词生成流程与单模板填写页完全一致
  - "完成此步骤"按钮：用户点击后，该 Step 标记为完成，下一个 Step 解锁

**页面逻辑**：
1. 用户进入工作流运行页，看到 Step 1 到 Step N 的列表
2. Step 1 默认可点击"开始"，其他 Step 为"待上一步完成"
3. 点击 Step 1 "开始" → 显示该 Step 对应的提示词填写界面
4. 用户完成填写、生成、复制后，点击"完成此步骤" → Step 1 标记为"已完成"
5. Step 2 的"开始"按钮自动解锁，用户继续
6. 所有 Step 完成后，页面顶部显示"🎉 工作流完成！"提示
7. 工作流的使用次数 +1（当用户完成至少一个 Step 时记录）

### 4.9 我的收藏 `/favorites`

**功能**：用户管理收藏的模板

**页面内容**：

### 4.13 模板管理（后台）`/admin/templates`

**功能**：管理员创建、编辑、发布、下线模板

**列表页内容**：
- 顶部："新建模板"按钮
- 表格列：标题、平台标签、内容类型标签、行业/场景标签、状态（草稿 / 已发布 / 下线）、使用次数、更新时间、操作（编辑 / 发布 / 下线 / 删除）
- 状态标签使用色块区分：草稿（灰色）/ 已发布（绿色）/ 下线（红色）

**编辑器页面（新建/编辑模板）**：
- 基础信息：
  - 标题（必填）
  - 一句话描述（必填，100 字以内）
- Prompt 模板内容（五段式，每段独立输入框）：
  - **Role**：角色定义（必填）→ 如："你是一名资深小红书运营专家"
  - **Goal**：目标说明（必填）→ 如："帮助用户撰写小红书种草笔记"
  - **Input**：变量定义（必填）→ 管理员在此段中用 `{{变量名}}` 语法定义变量，如：`产品：{{产品名称}}，核心卖点：{{核心卖点}}`
  - **Output**：输出要求（必填）→ 如："输出一篇 300 字以内的小红书笔记，包含标题、正文、标签"
  - **Example**：示例效果（必填）→ 管理员手写一段理想输出样例，帮助用户判断模板质量
- 变量自动识别 & 填写提示配置：
  - 编辑器在 Input 段下方实时显示识别到的变量清单（提取 Input 段中所有 `{{变量名}}` 并去重）
  - 每个变量名旁边有一个输入框，供管理员填写"变量填写提示"（可选，如留空则在用户端仅显示变量名）
  - 示例展示：
    ```
    识别到的变量：
      产品名称  [请输入产品完整名称，包括品牌名]
      核心卖点  [列出2-3个最吸引用户的卖点，用逗号分隔]
    ```
  - 如未识别到任何变量，给出提示"Input 段中没有找到变量，请使用 {{变量名}} 语法"
  - 变量填写提示存储为 JSON 格式：`{"产品名称": "请输入产品完整名称...", "核心卖点": "列出2-3个..."}`
- 标签配置：
  - **平台**：多选，从预定义列表中选择
  - **内容类型**：多选，从预定义列表中选择
  - **行业/场景**：多选，从预定义列表中选择
- 状态：草稿 / 发布 / 下线（可切换）
- "保存草稿"按钮 / "保存并发布"按钮

**编辑器页面逻辑**：
1. 保存时校验：标题、一句话描述、五段式所有段落均不能为空
2. 变量识别：每次 Input 内容变更时，用正则 `{{(.*?)}}` 提取所有变量名并去重显示，同时动态生成"填写提示"输入框
3. 变量提示同步：保存时将 `variable_hints` 以 JSON 格式写入数据库；如 Input 段变更导致变量消失，对应的提示也随之失效
4. 草稿状态的模板不出现在用户端
5. 已发布的模板可以被"下线"，下线后用户端不再显示（但不删除数据，使用次数保留）
6. 下线的模板可以重新发布

### 4.14 工作流管理（后台）`/admin/playbooks`

**功能**：管理员创建、编辑、发布工作流 Playbook（将多个模板按顺序组合为一个完整流程）

**列表页内容**：
- 顶部："新建工作流"按钮
- 表格列：工作流名称、步骤数量、状态（草稿 / 已发布 / 下线）、使用次数、更新时间、操作（编辑 / 发布 / 下线 / 删除）

**编辑器页面（新建/编辑工作流）**：
- 基础信息：
  - 工作流名称（必填）
  - 一句话描述（必填，100 字以内）
- **步骤配置（核心）**：
  - 按顺序添加/删除/排序步骤
  - 每个 Step 配置：
    - Step 序号（自动生成：1、2、3...）
    - Step 名称（必填，如"账号定位分析"）
    - 关联模板（必填，从已发布的模板中下拉选择）
    - Step 说明（可选，简要描述这一步做什么）
- 状态：草稿 / 发布 / 下线

**编辑器页面逻辑**：
1. 保存时校验：工作流名称、一句话描述、至少有 1 个步骤、每个 Step 必须关联一个模板
2. 草稿状态不出现在用户端
3. 已发布的工作流可以下线，下线后用户端不再显示
4. 工作流中引用的模板被删除或下线时，系统给出提示："该工作流中的部分模板已不存在，请检查"

**页面内容**：

  - **平台标签**：小红书、抖音、微信、公众号...
  - **内容类型标签**：种草、干货、带货、教程...

### 4.17 激活码管理（仅超管）`/admin/codes`

**功能**：超级管理员生成和管理用户激活码

- 激活码列表：
  - 激活码（格式：`KL7X-92A3-BM4R`，大写字母+数字，4-4-4）
  - 状态：未使用（绿色）/ 已使用（蓝色）/ 已禁用（灰色）

## 5. 设计语言规范

- **配色**：黑白灰为主色，搭配一个克制的强调色（用于主按钮、状态标签、关键数据）
- **字体**：系统默认字体，确保中文显示友好
- **布局**：左侧固定导航，内容区右侧展示
- **状态标签**：草稿（灰色）/ 已发布（绿色）/ 下线（红色），用色块区分
- **移动端**：确保平板和手机端可用（左侧导航在小屏幕上变为汉堡菜单）

## 7. API 接口清单

### 7.1 认证与激活

| 方法 | 路径 | 功能 | 权限 |
|---|---|---|---|
| POST | `/api/activation/verify` | 验证激活码是否有效 | 公开 |
| POST | `/api/activation/activate` | 使用激活码创建用户账号 | 公开 |
| POST | `/api/auth/login` | 用户名密码登录 | 公开 |
| POST | `/api/auth/logout` | 退出登录 | 已登录用户 |
| PUT | `/api/auth/password` | 修改密码（需要输入旧密码） | 已登录用户 |
| GET | `/api/auth/me` | 获取当前用户信息 | 已登录用户 |

### 7.2 模板相关

| 方法 | 路径 | 功能 | 权限 |
|---|---|---|---|
| GET | `/api/templates` | 获取模板列表（支持搜索、标签筛选、排序、分页，仅返回 published 状态的模板给普通用户） | 已登录用户 |
| GET | `/api/templates/:id` | 获取模板详情（含五段式内容、标签、使用次数、variable_hints） | 已登录用户 |
| POST | `/api/templates` | 创建新模板 | 管理员/超管 |
| PUT | `/api/templates/:id` | 更新模板（编辑内容、修改标签、切换状态，包括 variable_hints 更新） | 管理员/超管 |
| DELETE | `/api/templates/:id` | 删除模板（软删除，更新 status 为 deleted，保留数据） | 管理员/超管 |
| POST | `/api/templates/:id/use` | 记录使用一次（使用次数 +1，由前端在点击"生成提示词"时调用） | 已登录用户 |

### 7.3 工作流相关

| 方法 | 路径 | 功能 | 权限 |

## 8. 认证与安全

### 8.1 Token 机制

- **类型**：JWT (JSON Web Token)
- **存储方式**：前端 localStorage（用户登录成功后由前端写入）
- **传输方式**：`Authorization: Bearer <token>` Header（前端请求拦截器统一注入）
- **有效期**：
  - 用户勾选"记住我"：30 天
  - 用户不勾选"记住我"：浏览器关闭即失效（前端只把 Token 写入 sessionStorage 或加 expiresAt 软到期）
- **Token 内容**：包含 `user_id`、`role`、`iat`（签发时间）、`exp`（过期时间）
- **验证方式**：
  - 每次请求后端验证 JWT 签名和有效期
  - 同时检查 `users.is_active` 是否为 true（被禁用的用户即使 Token 未过期也无法访问）
- **登出**：前端清空 localStorage 中的 Token；后端将 `jti` 写入 Redis 黑名单，直到原 Token 自然过期

### 8.2 密码存储

- 使用 bcrypt 或同等强度的哈希算法，不存储明文密码
- 密码强度建议：至少 6 位（初期可放宽，后期可加强）

### 8.3 权限控制

- 后端中间件统一拦截：
  - `/api/admin/*` 路径：验证 role 为 admin 或 super_admin
  - `/api/admin/activation-codes/*`、`/api/admin/admins/*`、`/api/admin/ai-platforms/*`：验证 role 为 super_admin
  - 其他 `/api/*` 路径：验证已登录（有效 Token）
- 前端根据用户角色隐藏对应导航项（前端仅做体验优化，真正的权限控制在后端）

## 9. 系统初始化

### 9.1 初始化超级管理员

- 在系统配置文件（或环境变量）中配置初始超级管理员的用户名和密码

  - **industry**（行业/场景）：电商、教育、餐饮

> **v3 核心设计总结**：
> - **产品定位**：管理员发布模板和工作流 → 付费用户持激活码激活账号 → 按工作流步骤逐步填空生成提示词 → 复制到第三方 AI 平台使用
> - **核心用户体验**：工作流（多步骤流程）为主，独立模板为辅；用户进入首页看到推荐工作流，点击进入步骤列表，逐步完成创作
> - **工作流（Playbook）**：由多个步骤组成，每个步骤关联一个模板；步骤按顺序执行，每步独立生成提示词（极简版，不做变量传递）
> - **模板**：采用"五段式"统一结构，配合 `{{变量名}}` 自动识别表单字段；新增 `variable_hints` 字段，管理员可为每个变量配置填写提示
> - **标签体系**：完全动态配置（平台 / 内容类型 / 行业场景 三维标签），管理员在"标签管理"后台自由增删、排序
> - **用户端体验**：首页（推荐工作流 + 热门模板）→ 工作流列表 / 模板库 → 工作流步骤列表 → 填表 → 生成可编辑 Prompt → 复制 → 跳第三方 AI 平台
> - **使用次数统计规则**：点击"生成提示词"按钮时 +1
> - **模板/工作流删除策略**：软删除（保留数据，status 标记为 deleted/archived）
> - **用户准入**：以"激活码"作为用户准入机制，控制谁能使用平台；激活码由超管在后台生成和管理
> - **后台管理（管理员）**：模板管理 / 工作流管理 / 标签管理 / 数据看板
> - **后台管理（仅超管）**：激活码管理 / 管理员账号 / AI 平台配置
> - **初始 AI 平台列表**：DeepSeek、Kimi、豆包、通义千问
